How Matching Works

v6 ordered waterfall engine.

Step 1 — Score 100
Pol + Endo + Prem + Name + Date
All five signals agree. Maximum confidence.
Step 2 — Score 97
Pol + Endo + Prem + Name
All four strong signals, no date check.
Step 3 — Score 95
Pol + Prem + Name + Date
No endo match, but pol+prem+name+date all agree.
Step 4 — Score 92
Pol + Prem + Name
Good signal without date.
Step 5 — Score 90
Endo + Prem + Name + Date
Strong endorsement match.
Step 6–8
Endo+Prem+Name / Pol+Prem / Endo+Prem
Two-signal combos. Scores 87–82.
Step 9–10
Name+Prem+Date / Name+Prem
Used when polno=0 in master or polno not available.
Step 11–12
Pol only / Endo only
Single signal — low confidence (50–55). May show as Partial.
Zero Commission
Commission between -₹1 and ₹1
No matching attempted.
Contra
Sum of commission for same polno ≈ 0
Offsetting entries. If Cno found = Partial Match with score; if not = Contra.
Premium Logic (Any mode)

Checks: Total vs Total, OD vs Total, TP vs Total, OD+yearDiff×TP vs Total (multi-year), group sum vs master PBST, Share%-adjusted TP/OD/Total, and OD vs OD, TP vs TP directly. Any one passing within ±2% (or ±₹50) = premium match.

Policy No Trimming

Before comparing, both statement and master policy numbers are trimmed to the same length from the specified direction. Colon (:) is stripped first (Excel behaviour). This handles format differences like 4225/1000512039/00/0000 vs 4225/1000512039/00/00 without needing exact length matching. Contains-check also applied after trimming so partial matches still work.
